The three host renderers

Vulkan compositor MOST CHANGED

Our default renderer · native libwinlator / VulkanRendererContext.cpp

  • NEW · P1
    Real spatial upscaling — SGSR 1.0
    single-pass upscaler · from Qualcomm / Snapdragon Game Studios — snapdragon-gsr (BSD-3)
  • NEW · P1
    Real spatial upscaling — FSR 1.0 (EASU + RCAS)
    two-pass upscale + sharpen · from AMD GPUOpen / FidelityFX — FidelityFX-FSR (MIT)
  • NEW · P1
    FSR-Fit (aspect-preserving) + None / Linear / Nearest
    live "Scaling mode" picker in the in-game drawer · aspect mode reimplemented from GameNative approach
  • UPD · P2
    Post-process effects now work on Vulkan
    FXAA · CRT · Toon · NTSC · Color (brightness/contrast/gamma) · fake-HDR look — ported from our own GL effects (were GL-only / grayed-out)
  • UPD · P3
    Data-driven effect engine
    load arbitrary .fx presets instead of hardcoded effects · approach from ReShade / GameHub
Why this matters: today the Vulkan renderer offers no post-process and only sampler-based scaling. After this it becomes the only renderer in any fork with both SGSR and FSR1 upscaling plus the full effect suite — on the default, fastest path.

OpenGL renderer LEANER

Compatibility path · GLRenderer + EffectComposer

  • KEEP
    Existing full effect suite stays
    SGSR(=CAS sharpen) · fake-HDR · FXAA · CRT · Toon · NTSC · Color — already in our tree (Winlator / Star-Bionic lineage)
  • UPD · P4
    Lean native-GLES2 rework
    reduce Java GLSurfaceView overhead, cheaper blit/upscale — philosophy from GameHub 5.3.5's native GLES2 X-server (clean-room, no code taken)
Note: upscaling (SGSR/FSR) is being added to the Vulkan path first. The GL path keeps its existing CAS "SGSR"; real FSR/SGSR on GL is possible later but is not in current scope.

SurfaceFlinger / ASR renderer UNCHANGED

Low-latency direct-scanout · ASurfaceRenderer (experimental, gated)

  • KEEP
    Direct SurfaceControl scanout
    this is our port of GameHub 5.3.5 "Native Rendering+" — already the lean low-latency win others lack
Limitation (by design): ASR bypasses the Vulkan compositor draw pass, so the new upscalers and ported effects do not apply in this mode. It stays a pure low-latency passthrough.
